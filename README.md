## Lo que he construido

### 🗂️ planta_remar — App de gestión operativa (en uso real)

App móvil + web para la gestión de tareas y workflows de un negocio con varias sedes, hoy en uso por su equipo. La pensé, diseñé, desarrollé y desplegué entera: autenticación, roles, notificaciones push/WhatsApp, y registro de tareas con evidencia fotográfica y trazabilidad — base para sistemas de gestión de calidad. Diseñada como producto reutilizable para más clientes, no como apaño puntual.

**Stack:** React Native/Expo · Next.js · Firebase (Firestore/Auth/FCM/Functions) · Turborepo/pnpm · Vercel

### 📦 inventario-remar — SaaS de inventario en producción

Plataforma full-stack de inventario multi-centro, multi-usuario y desplegada en producción. Tipado robusto (uniones discriminadas en TypeScript), reglas de seguridad por rol, borrado en cascada atómico y una revisión sistemática de código antes de cada despliegue. Aquí aprendí a enviar a producción con criterio: seguridad, rendimiento y mantenibilidad.

**Stack:** Next.js · Firestore · Firebase Auth · TypeScript · Turborepo/pnpm · Vercel

### 🤖 conciliacion-mcp — Agente de IA sobre datos reales

Servidor Model Context Protocol (MCP) que deja a un agente LLM conciliar movimientos bancarios contra exportaciones de un ERP. Parsea extractos de tres bancos y un libro auxiliar, ejecuta el algoritmo de emparejamiento y genera un informe. Es mi laboratorio para meter IA agéntica en flujos reales, no en demos.

**Stack:** Python · MCP · parsing de Excel/PDF · lógica de matching

### 📄 DocuStrike — Extracción documental con visión por LLM *(repo privado)*

Pipeline de IDP/OCR que extrae datos estructurados de registros manuscritos usando visión por LLM (Gemini Vision). Lo construí con un framework propio de evaluación: ground truth, categorización de errores, post-procesado con regex e iteración de prompts versionada. Sé probar hipótesis con IA y medir si de verdad funcionan.

**Stack:** Python · Gemini Vision · framework de evals · salida a Excel
